# Frontend Requirements (React)

## Dashboard (UI + UX)

- Create a **responsive dashboard layout** that works on **desktop and mobile**.
- Provide clear **navigation** between dashboard pages/views.
- Use modern React patterns (**functional components + hooks**).

## Data visualization (interactive charts)

Display interactive charts using a chart library such as **Recharts** or **Chart.js**:

- **Bar Chart** — visualize **monthly enrollment trends**
- **Pie Chart** — show **distribution of students across different courses**
- **Line Chart** — display **attendance patterns over school days**

## Data fetching

- Implement **data fetching** for dashboard data.
- Include **loading states** while fetching.
- Include **error handling** and user-friendly error messages.

## Weather widget (API integration)

Integrate a real-time weather API (e.g., OpenWeatherMap / WeatherAPI / similar):

- Show **current conditions** (e.g., temperature, humidity, wind)
- Show a **forecast** (e.g., 5‑day)
- Support **location-based search** (city name or geolocation)
- Provide a **visual weather representation** (icons)
- Handle **API rate limiting** and errors gracefully

## Quality & maintainability (frontend)

- Follow React best practices (hooks, component composition)
- Implement proper **error boundaries** where appropriate
- Keep code clean and maintainable
